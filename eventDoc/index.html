Events to listen

verify User
This event ("verify-user") has to be listened on the user's end to verify user authentication.
user will only be set as online after verification of authentication token.
online-user-list event
This event ("online-user-list") has to be listen on user's end to get updated online user list .
and the online-user-list will be in the form of object(userId:userName);
error-message event
This event ("error-message")has to listen on both user's end so if any error occured on socket then error will be notified to user
userId event
This event ("userId") has to be listened on user side . whenever user perform operation like send friend request or reject friend request or unfriend someone or accept request at the same time this event calls and user will get update everytime whenever these four operation performs.
searched-result event
This event ("searched-result") has to be listened on user side . whenever user search any name then all matched name will be displayed on screen.
friend-notify event
This event ("friend-notify") has to be listened on friend user side . whenever user edit/update/delete/undo any task or subtask of friend at same time all friend will get notification.
get-update event
This event ("get-update") has to be listened on user side . whenever user create/edit/update/delete/undo any task or subtask at same time this event will give updated task list.

Events to emit

set-user event
This event ("set-user") has to be emitted when user get verify .
if user authorized using authToken then user will be added to online-list.
create-task event
This event ("create-task") has to be emitted when user creates task-title.
add-task event
This event ("addtask") has to be emitted when user creates task.
add-sub-task event
This event ("add-sub-task") has to be emitted when user creates new sub-task
update-task event
This event ("update-task") has to be emitted when user updates any task
update-sub-task event
This event ("update-sub-task") has to be emitted when user updates any sub task.
completed-task event
This event ("completed-task") has to be emitted when user marks any task as completed.
completed-sub-task event
This event ("completed-sub-task") has to be emitted when user marks any sub-task as completed.
re-complete-task event
This event ("re-complete-task") has to be emitted when user marks any task as not completed.
re-complete-sub-task event
This event ("re-complete-sub-task") has to be emitted when user marks any sub-task as not completed.
delete-full-task event
This event ("delete-full-task") has to be emitted when user deletes any task.
delete-task event
This event ("delete-task") has to be emitted when user deletes any task.
delete-sub-task event
This event ("delete-sub-task") has to be emitted when user deletes any sub-task.
delete-friend-task event
This event ("delete-friend-task") has to be emitted when user deletes any friend's task.
delete-friend-sub-task event
This event ("delete-friend-sub-task") has to be emitted when user deletes any friend's sub-task.
search-user event
This event ("search-user") has to be emitted when user searches any friend name.
send-request event
This event ("send-request") has to be emitted when user sends request to any person.
reject-request event
This event ("reject-request") has to be emitted when user rejects any friend request.
accepted-request event
This event ("accepted-request") has to be emitted when user accepts any friend request
send-to-all-friend event
This event ("send-to-all-friend") has to be emitted when user perform any operation on friend's task at same this event send notification to all friends.
undo-task event
This event ("undo-task") has to be emitted when user want to recover recently deleted task
undo-sub-task event
This event ("undo-sub-task") has to be emitted when user want to recover recently deleted sub-task
undo-friend-task event
This event ("undo-friend-task") has to be emitted when user deletes any task of friend and if user wants deleted task of friend then this event will call.
undo-friend-sub-task event
This event ("undo-friend-sub-task") has to be emitted when user deletes any sub task of friend and if user wants deleted sub task of friend then this event will call.
unfriend event
This event ("unfriend") has to be emitted when user unfriend someone.
disconnection
This event ("disconnection") has to be emitted, when the user gets disconnected.
